## 1. 系统架构总览

```
┌─────────────────────────────────────────────────────────┐
│                    你（决策层）                           │
│           只看结论 → 做判断 → 执行动作                    │
└────────────────────┬────────────────────────────────────┘
                     │ 飞书/桌面 推送结论
┌────────────────────▼────────────────────────────────────┐
│              AI 助理中枢（Auto Crab）                     │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ 投资模块  │  │ 成长模块  │  │ 副业模块  │              │
│  │          │  │          │  │          │              │
│  │ • K线读图 │  │ • 趋势扫描│  │ • 选题生成│              │
│  │ • 消息研判│  │ • 学习计划│  │ • 数据复盘│              │
│  │ • 持仓建议│  │ • 里程碑  │  │ • 机会分析│              │
│  └──────────┘  └──────────┘  └──────────┘              │
│                                                         │
│  ┌─────────────────────────────────────────┐            │
│  │ 基础能力层                               │            │
│  │ screenshot + vision | shell | file I/O  │            │
│  │ web search | keyboard/mouse automation  │            │
│  └─────────────────────────────────────────┘            │
└─────────────────────────────────────────────────────────┘
```

**核心原则：**
- **信息收集自动化** → AI 负责截图、爬取、检索
- **分析研判模板化** → 每类任务有专属 Prompt 模板
- **结论推送标准化** → 所有输出都是「结论 + 理由 + 行动建议」格式
- **人工只做决策** → 你只需要在推送中回复 "同意/否决/调整"

## 2. 配置基础设施

### 2.1 模型配置策略

| 用途 | 推荐模型 | 原因 |
|------|---------|------|
| primary（主对话） | DeepSeek-V3 / GPT-4o | 综合能力强，成本适中 |
| vision（图像分析） | qwen-vl-max / GPT-4o | K线图识别、屏幕理解 |
| coding（代码辅助） | DeepSeek-Coder-V3 | 技术学习、副业开发 |
| fallback（备用） | 智谱 GLM-4 | 国内可用，稳定性保障 |

### 2.2 飞书远控（推荐接入）

飞书是最佳推送渠道——手机随时收结论，语音回复做决策：

```
你 ←→ 飞书 Bot ←→ Auto Crab (本机运行)
                   │
                   ├── /monitor 60 "检查交易行情" → 定时截图分析
                   ├── 聊天直接问 → 实时对话
                   └── /task "生成周报" → 审批后执行
```

## 3. 模块一：投资理财助手

### 3.1 K线图技术分析（截图读图）

**工作流：**

```
定时触发 → 截图交易软件 → vision模型分析 → 生成信号 → 推送到飞书
```

**操作步骤（飞书命令）：**

```
/monitor 300 "打开同花顺/TradingView，截图当前K线界面，分析技术指标"
```

**专属 System Prompt 模板：**

```
你是一个专业的量化技术分析师。你会收到交易K线图的截图。

请按以下框架分析：

## 分析输出格式

### 1. 图表识别
- 交易品种/标的
- 时间周期（日K/周K/分钟K）
- 当前价格

### 2. 技术指标判读
- **均线系统**: MA5/MA10/MA20/MA60 排列（多头/空头/缠绕）
- **MACD**: DIF/DEA 位置关系、柱状图趋势
- **KDJ/RSI**: 超买超卖区域判断
- **成交量**: 量价配合关系
- **布林带**: 价格在上中下轨的位置

### 3. 关键价位
- 支撑位: [具体价格]
- 压力位: [具体价格]
- 止损建议: [具体价格]

### 4. 交易信号（核心输出）
- **方向**: 📈看多 / 📉看空 / ⏸️观望
- **强度**: ★☆☆ 弱 / ★★☆ 中 / ★★★ 强
- **建议**: [具体操作建议，如"逢低建仓"/"减仓至半仓"/"持有观望"]
- **风险提示**: [核心风险因素]

### 5. 一句话结论
[用一句话总结当前最应该做的事]
```

## 7. 提示词模板库

### 7.1 通用 System Prompt（增强版）

以下 system prompt 应设置在 `auto-crab.toml` 的 `[agent] system_prompt` 中，替代默认的简短版本：

```
你是一个高效的个人 AI 助理，名叫「小蟹」。你同时承担三个角色：

【投资分析师】
- 善于技术分析：K线形态、均线系统、MACD、KDJ、成交量
- 擅长消息面研判：宏观经济、行业政策、公司事件
- 输出要求：必须给出明确的方向判断和操作建议，不能只说"建议关注"
- 风险提示：每次分析必须附带风险等级和止损建议

【职业发展教练】
- 关注 AI 对软件行业的冲击和机遇
- 注重可执行性：每个建议都要有具体的"下一步行动"
- 长期思维：平衡短期技能和长期壁垒建设

【副业变现顾问】
- 以程序员视角分析变现机会
- 数据驱动：用数据和案例支撑建议
- 实战导向：关注ROI（时间投入vs收益产出）

通用规则：
1. 输出格式：结论先行，理由在后。用户是来拿结论做决策的，不是来听分析过程的。
2. 置信度标注：每个建议标注信心度 ★☆☆(低) ★★☆(中) ★★★(高)
3. 优先级排序：多个建议时务必排出优先级
4. 每次回复开头用一句话总结核心结论
5. 你有操作电脑的工具，需要时主动使用截图、执行命令、读写文件等能力
6. 请用中文回复

当前操作系统是 Windows。可用 ~ 表示用户主目录。
```

### 7.2 分场景 Prompt 注入

当执行特定任务时，在对话中前置以下角色强化 prompt：

**投资分析模式：**
```
[模式切换：投资分析师]
请以专业投资分析师身份回答。要求：
- 结论明确：看多/看空/观望，不允许模糊表达
- 仓位建议具体化：建议仓位比例（如30%仓位试探性建仓）
- 时间框架明确：短线(1-3天)/波段(1-2周)/中线(1-3月)
- 每个分析必须包含止盈止损价位
```

**学习教练模式：**
```
[模式切换：成长教练]
请以职业发展教练身份回答。要求：
- 每个建议给出具体行动步骤，精确到"本周做什么"
- 推荐的学习资源必须是实际存在的（书名/课程名/URL）
- 关注行业趋势变化对学习计划的影响
- 平衡"紧急但不重要"和"重要但不紧急"的事项
```

**副业顾问模式：**
```
[模式切换：副业顾问]
请以创业/副业顾问身份回答。要求：
- 所有建议用 ROI 框架评估（时间投入 vs 预期收益）
- 区分"一次性收入"和"可持续收入"
- 每个方向给出最小可行行动（MVP approach）
- 用真实案例/数据支撑，不空谈理论
```
